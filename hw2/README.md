# Семинар 2: Моделирование данных с использованием E/R диаграмм

## Описание задания

В этом задании вам нужно будет создать E/R диаграммы для библиотечной системы и моделирования отношений. Используйте *
*PlantUML** для создания диаграмм. Все файлы должны быть размещены в папке `src/` в соответствие с заданием.

Пример использования PlantUML для создания E/R диаграмм в файле `./src/example.puml`. 
---

### Задача 1: Библиотечная система

Нарисуйте E/R диаграмму для библиотечной системы на основе следующих требований:

В библиотеке хранятся книги (издания). Каждая книга (издание) имеет уникальный идентификатор, название, автора, год
издания и количество страниц. Книги издаются издательствами, у которых есть название и адрес. В библиотеке есть
экземпляры книг (физические/электронные копии издания). Каждый экземпляр имеет уникальный идентификатор и положение на
полке. Экземпляр книги связан с конкретной книгой. Книги могут принадлежать одной или нескольким категориям. Категории
могут быть вложенными (одна категория может быть подкатегорией другой). У каждой категории есть название. В библиотеке
есть читатели. Каждый читатель имеет уникальный идентификатор, имя, фамилию, адрес и дату рождения. Читатель может брать
книги на определенный срок. Для каждой выдачи книги записывается дата, когда книга была взята, планируемая дата возврата
и фактическая дата возврата (если книга уже возвращена).

Ключи связи не указываются, связь отображается между сущностями, а не атрибут связи -> уникальный идентификатор.

**Файл для выполнения:** `src/library.puml`

---

### Задача 2: Моделирование отношений

Смоделируйте следующие отношения в E/R. Используйте только те сущности, которые указаны в задании. Не добавляйте новые,
а так же не используйте атрибуты.

1. **Адреса**:
    - Квартира находится в доме.
    - Дом расположен на улице.
    - Улица находится в городе.
    - Город находится в стране.

**Файл для выполнения:** `src/relationships_address.puml`

2. **Спортивные матчи**:
    - В системе хранится информация о матчах между командами.
    - Каждый матч проходит между двумя командами под руководством одного арбитра.
    - Команда не играет сама с собой и может участвовать в не ограниченном числе матчей
    - Арбитр может судить разные матчи.

**Файл для выполнения:** `src/relationships_match.puml`

3. **Родственные связи**:
    - У каждого человека есть два биологических родителя: отец (мужчина) и мать (женщина).

**Файл для выполнения:** `src/relationships_parents.puml`

## Инструкция по выполнению

- **9 баллов**: Есть незначительные замечания (например, направление в схеме перепутано).
- **8 баллов**: Задание сдано вовремя, но была одна переделка после дедлайна, и всё исправлено правильно. Или есть
  ошибки, но в целом задание выполнено.
- **7 баллов**: Задание сдано не вовремя, есть ошибки, но в целом сделано.
- **6-5 баллов**: Сделано меньше половины задания.
- **Ниже 5 баллов**: Задание отправлено на доработку.
