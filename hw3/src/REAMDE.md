# Решение задания #3

## Часть 1: Теоретические вопросы

### 1. Почему любое отношение в реляционной схеме имеет по крайней мере один ключ?
Потому что отношение — это множество уникальных строк. Даже если нет явных полей, вся строка целиком уже является суперключом. 
Из него всегда можно выделить минимальный уникальный набор полей — это и будет ключ.

### 2. Почему мы используем промежуточные таблицы для связи "многие ко многим"?
Потому что реляционная модель не позволяет хранить несколько ссылок в одном поле. 
Промежуточная таблица с двумя внешними ключами решает проблему: каждая её строка связывает 
одну запись из первой таблицы с одной записью из второй, позволяя создавать любые множественные связи.

### 3. В каких случаях оправдано использование автоинкремента, а в каких лучше выбрать первичный ключ из сущности или использовать составные ключи?
Автоинкремент лучше использовать когда: нет естественного ключа, который уникален или состоит из нескольких полей, 
а естественный ключ лучше, когда есть стабильное уникальное поле или составной ключ логически точно идентифицирует запись. 
Составной ключ для промежуточных таблиц связи М:М и ситуаций, где комбинация полей — единственная гарантия уникальности
